# Cybersecurity Portfolio Website

A professional, single-page portfolio website for Aditya Badarpuria - Aspiring Penetration Tester & SOC Analyst.

## Features

- **Modern Dark Theme**: Cybersecurity-themed design with dark backgrounds and blue accents
- **Particle.js Animation**: Interactive particle background in the hero section
- **Responsive Design**: Fully responsive layout for desktop, tablet, and mobile devices
- **Smooth Animations**: Hover effects, card transitions, and smooth scrolling
- **Single File Structure**: Easy to deploy on GitHub Pages

## Sections

1. **Hero/Banner**: Eye-catching introduction with animated particles and rotating role descriptions
2. **About**: Professional summary and background
3. **Education**: Academic qualifications (MSc Cyber Security & BCA)
4. **Skills**: Organized categories of technical skills and security expertise
5. **Projects**: Showcase of key projects with tech stacks
6. **Focus Areas**: Primary areas of interest in cybersecurity
7. **Contact/Footer**: Social media links and contact information

## Deployment on GitHub Pages

### Option 1: Direct Repository Deployment

1. Create a new repository on GitHub named `<your-username>.github.io`
2. Upload all files from this project to the repository
3. Your website will be live at `https://<your-username>.github.io/`

### Option 2: Project Repository

1. Create a new repository on GitHub (e.g., `portfolio`)
2. Upload all files from this project
3. Go to Settings → Pages
4. Select the main branch as the source
5. Your website will be live at `https://<your-username>.github.io/portfolio/`

## Local Testing

To test the website locally:

1. Simply open `index.html` in your web browser
2. Or use a local server:
   ```bash
   # Using Python 3
   python -m http.server 8000
   
   # Using Node.js http-server
   npx http-server
   ```
3. Open `http://localhost:8000` in your browser

## Customization

To customize the portfolio:

1. **Personal Information**: Update the content in the HTML sections
2. **Social Links**: Replace GitHub and LinkedIn URLs with your own
3. **Images**: Replace images in the `images/` folder with your own
4. **Colors**: Modify the CSS color variables in the `<style>` section
5. **Sections**: Add or remove sections as needed
